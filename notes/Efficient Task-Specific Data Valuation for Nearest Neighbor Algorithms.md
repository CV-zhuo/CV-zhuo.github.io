# 2.问题

* 如何公平地把酬劳分配给提供数据的每一个贡献者，以真实反映他们提供的这条数据的“价值”？
* 当前方法：计算复杂度呈指数级爆炸，无法落地。

# 3.创新点与方法

* 首次将机器学习中计算Shapley Value的复杂度从指数级降到了拟线性时间。
* 首次将数据库领域的近似搜索技术（LSH）引入到博弈论的数据定价中，实现了亚线性的计算速度。

# 5.局限

* 该算法仅适用于 KNN 族算法，利用的是KNN特殊的局部性。对于深度学习网络、随机森林等，该公式不成立。在实际应用中，只能先用深度模型提取特征，再用KNN做替代来估算数据价值，这存在一定的近似偏差。
* 复杂变体的复杂度依然很高。
* LSH近似算法对数据集的内在几何属性很敏感。如果数据分布密集、区分度低，比如特征全都糊在一起，LSH带来的加速效果就会大打折扣，甚至不如精确算法。并且LSH需要仔细调节哈希表数量、桶宽等超参数。


1.先将数据集通过神经网络提取特征；  
2.使用LSH划分出子集；  
3.对子集中的数据计算到测试点的L2距离；  
4.由近到远排序；  
5.判断标签对错；  
6.从最远的点开始，用递推公式倒推每个点对测试点的SV；  
7.验证总和是否为模型的预测准确率
