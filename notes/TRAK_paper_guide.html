❓ 用生活例子理解"数据归因"

🍎 类比：老师批改作文

想象你是一个AI老师，批改了10000篇作文后学会了"什么是好作文"。

现在有一篇新作文，你给了它高分。

数据归因要解决的问题就是：你给高分，到底是因为你读过哪些好作文？哪些训练作文对你的这个判断影响最大？

🎬 更贴切的类比：电影推荐算法

Netflix 推荐你看《权力的游戏》。数据归因能告诉你：
这个推荐是因为你看过《冰与火之歌》原著？还是因为你喜欢《指环王》？还是因为你评了《黑暗荣耀》高分？

找到"最负责任的"训练数据，就是数据归因的目标。

📏 怎么评价一个方法好不好？

论文提出了一个新的评分标准：Linear Datamodeling Score (LDS)

📐 理解 LDS 的核心思想

核心思维：预测力

如果你的归因方法是对的，那么它应该能预测：如果我把某些训练数据删掉，模型的表现会怎么变化？

就好比：你说训练数据X对预测结果影响最大。那我们真的把X删掉重新训练，模型表现确实变差了——这说明你的归因是对的。

LDS 分数越高 = 方法的"预测力"越强 = 归因越准确

🧩 每个步骤的直觉理解

步骤1：线性化（为什么要这么做？）

💡 类比：用切线近似曲线

神经网络非常复杂（像一条弯曲的曲线）。在某个局部，我们可以用一条直线来近似它（切线）。

这个近似让原来"复杂模型的数据归因问题"变成了"简单逻辑回归的数据归因问题"——后者是有成熟解法的！

① 原文是怎么说的？
📄 原文 Section 3.2

"The main roadblock to applying Definition 3.1 in our setting is that we are studying a non-linear model—that is, our model output function may not be a linear function of θ. We address this issue by approximating f(z; θ) with its Taylor expansion centered around the final model parameters θ*."

📝 译文：在我们的场景中，应用 Definition 3.1 的最大障碍是——我们研究的是非线性模型，模型输出函数不是 θ 的线性函数。我们用以 θ* 为中心的泰勒展开来近似 f(z; θ)，以此解决这个问题。
📄 原文公式（Section 3.2, Step 1）
f̂(z; θ) := f(z; θ*) + ∇θf(z; θ*)T(θ − θ*)
📝 公式拆解：
  • f(z; θ*)：模型在当前最优参数 θ* 下的输出值（一个已知的数）
  • ∇θf(z; θ*)：模型输出对参数的梯度（即"斜率"，也已知）
  • (θ − θ*)：参数偏离 θ* 的距离
  → 整体含义：用"当前值 + 斜率 × 偏移量"来预测新位置的函数值，这就是切线近似
② 什么是"线性"和"非线性"？
📈 非线性（神经网络）

输入变化一点，输出以复杂、不可预测的方式变化。就像山路——弯弯绕绕，无法用一条直线描述。

❌ 没有通用的归因公式

📏 线性（逻辑回归）

输入变化多少，输出就按比例变化。就像高速公路——笔直，完全可预测。

✅ 有成熟的归因公式（Newton步近似）

🌍 生活类比：地图投影
地球是球形（非线性曲面），但一个城市的地图是平面（线性）——在局部范围内，用平面近似球面误差很小。神经网络的线性化也是同样的道理：在 θ* 附近的局部范围，用直线近似曲线。
③ 线性化之后，视角发生了什么变化？
📄 原文关键句

"Rather than viewing f(z; θ) as a non-linear model acting on inputs x, we can view it as a linear model acting on inputs ∇θf(z; θ*)."

📝 译文：与其把 f(z; θ) 看作"以原始数据 x 为输入的非线性模型"，不如把它看作"以梯度向量 ∇f 为输入的线性模型"。
🔄 视角转换对照表
　 线性化之前 线性化之后
模型类型 非线性神经网络 线性逻辑回归 ✅
"输入"是什么 原始数据 x（图片/文字像素） 梯度向量 ∇f（每个样本的梯度）✅
可用归因工具 ❌ 无成熟方法 ✅ Newton步近似（40年前就有的公式）
④ 为什么这个近似是合理的？
📄 原文解释

"Prior work (around, e.g., the empirical neural tangent kernel) suggests that this approximation can be relatively accurate, especially for overparameterized neural networks."

📝 译文：已有研究表明，这个近似相当准确，尤其对过参数化神经网络（即参数量远超训练数据量的大模型）来说更是如此。
💡 直觉解释：大型神经网络训练到收敛后，参数已经非常稳定，不再大幅变化。在这个"稳定区域"内，曲线非常平缓——用切线近似误差极小，就像平静湖面上，局部水面近似为平面完全没问题。
🔗 与 eNTK 的联系（原文 Note）：
原文还特别说明，Step 1 的线性化本质上就是用empirical Neural Tangent Kernel (eNTK)来近似神经网络。eNTK 是近年来理论分析神经网络泛化性的重要工具，TRAK 借用了这个工具来做归因。
⑤ 损失函数推导：为什么"输入"变成了梯度向量 gᵢ？

这是整个线性化最关键的一步推导，搞懂它就能彻底理解 TRAK 的核心逻辑。

① 原始损失函数是什么？
📄 原文 Equation (7)
θ*(S) = argmin Σ log[ 1 + exp(−yi · f(zi; θ)) ]
📝 每个符号的含义：
  • argmin：找到让后面式子最小的 θ 值（"最优参数"）
  • Σ：对所有训练样本求和（第1个损失 + 第2个 + … + 第n个）
  • yᵢ：第 i 个样本的标签，只有两个值：+1（正类）或 −1（负类）
  • f(zᵢ; θ)：模型对样本 i 的输出（一个实数，正数倾向正类，负数倾向负类）
  • yᵢ · f(zᵢ; θ)：乘积为正 = 预测正确；乘积为负 = 预测错误
  • log[1 + exp(−…)]：logistic loss，乘积越大（预测越准），损失越接近 0
💡 一句话理解：找一组参数 θ，让模型在所有训练样本上预测尽量准确（损失之和尽量小）。
② 定义两个新变量 gᵢ 和 bᵢ
// 变量1：梯度向量（新的"输入特征"）
gi := ∇θf(zi; θ*)　← 模型输出对参数的偏导，是一个向量

// 变量2：常数偏置项
bi := f(zi; θ*) − ∇θf(zi; θ*)ᵀ θ*　← 所有项都是已知常数
💡 bᵢ 为什么是常数？因为 bᵢ 里的每一项——f(zᵢ; θ*)、∇θf(zᵢ; θ*)、θ*——都是模型训练完毕后就固定下来的已知值，和"待求的 θ"无关。所以 bᵢ 是每个样本各自独立的常数偏置，不随 θ 变化。
③ 代入化简：展开括号 → 合并 → 得到 θᵀgᵢ + bᵢ
// 从泰勒展开公式出发
f̂(zᵢ; θ) = f(zᵢ; θ*) + ∇θf(zᵢ; θ*)ᵀ (θ − θ*)

// 展开括号
　　　　= f(zᵢ; θ*) + ∇θf(zᵢ; θ*)ᵀθ − ∇θf(zᵢ; θ*)ᵀθ*

// 用 gᵢ 替换梯度符号
　　　　= f(zᵢ; θ*) + giᵀθ − giᵀθ*

// 把不含 θ 的两项合并为 bᵢ
　　　　= f(zᵢ; θ*) − giᵀθ* + giᵀθ
　　　　　　↑ 这两项合并就是 bᵢ

　　　　= bi + giᵀθ

// 交换内积顺序（结果相同）
　　　　= θᵀgi + bi　✅ 化简完成
④ 把 θᵀgᵢ + bᵢ 代入损失函数
将 f(zᵢ; θ) 替换为 θᵀgᵢ + bᵢ，一步替换，损失函数变成：
Σ log[ 1 + exp(−yi · (θᵀgi + bi)) ]
⑤ 与标准逻辑回归对比——关键的一步
　 损失函数 "输入特征"
标准逻辑回归 Σ log[1+exp(−yᵢ·(θᵀxᵢ+bᵢ))] xᵢ（原始数据）
线性化后的神经网络 Σ log[1+exp(−yᵢ·(θᵀgᵢ+bᵢ))] gᵢ（梯度向量）
✅ 两个公式形式完全相同，唯一区别：xᵢ 换成了 gᵢ。
→ 原来无法处理的神经网络归因问题，现在变成了有 40 年成熟解法的逻辑回归归因问题！
⑥ 完整推导链总结
原始问题
神经网络损失：Σ log[1 + exp(−yᵢ · f(zᵢ; θ))]
↓ 泰勒展开：f(zᵢ; θ) ≈ f(zᵢ; θ*) + ∇θf(zᵢ; θ*)ᵀ(θ − θ*)
↓ 定义 gᵢ = ∇θf(zᵢ; θ*)，bᵢ = f(zᵢ; θ*) − gᵢᵀθ*
↓ 代入化简：f(zᵢ; θ) ≈ θᵀgᵢ + bᵢ
得到等价的逻辑回归
Σ log[1 + exp(−yᵢ · (θᵀgᵢ + bᵢ))]
↓ 套用 Newton 步近似公式（Definition 3.1）
输出归因分数
τ(z)ᵢ = g(z)ᵀ · (ΦᵀΦ)-1 · gᵢ · (1 − p*ᵢ)
⑥ 线性化带来的新问题（引出 Step 2）

⚠️ 线性化虽然解决了"非线性"问题，但带来了新麻烦：

线性化后，新的"输入"是梯度向量 ∇f。梯度的维度 = 模型参数量。

举个例子：BERT-base 有 1.1 亿个参数 → 每个样本的梯度向量有 1.1亿维！

→ 直接处理这么高维度的向量，计算量爆炸，根本无法运行。

✅ 这就是为什么需要 Step 2：用随机投影把维度压缩到可处理的大小（比如 4000 维）。

步骤2：降维（为什么需要这一步？）

💡 类比：压缩文件

现代神经网络有数百万个参数，意味着梯度向量也有数百万维。直接处理太慢。

随机投影（Johnson-Lindenstrauss 引理）保证：把高维向量压缩到低维后，向量之间的相对关系（内积）基本不变——就像压缩图片后仍然认得出来。

① 问题有多严重？数字说话

线性化之后，每个训练样本都有一个梯度向量 gᵢ，维度 = 模型参数量 p。当 p 很大时：

ResNet-18（ImageNet）：p ≈ 1100万　→ gᵢ 是 1100万维向量
BERT-base（QNLI）　：p ≈ 1.1亿　 → gᵢ 是 1.1亿维向量
GPT-3　　　　　　　：p ≈ 1750亿　→ gᵢ 是 1750亿维向量

⚠️ 实际计算有多难？以 BERT + QNLI 为例：

训练集 n = 50000 个样本，每个梯度向量 1.1亿维
→ 梯度矩阵 Φ 大小 = 50000 × 1.1亿 = 5.5万亿个数字，约 44 TB
→ 归因公式需要算 (ΦᵀΦ)⁻¹，这是 1.1亿 × 1.1亿 的矩阵求逆
→ 计算量 ≈ O(p³) = O((1.1亿)³) ← 完全不可能完成 ❌

② 随机投影：做法极其简单

核心操作：生成一个随机矩阵 P，把每个高维梯度向量乘以它，直接压缩到低维。

// P 是随机矩阵，大小 p × k，每个元素从 N(0,1) 随机采样
P ∈ ℝp×k，　p = 原始维度（1.1亿），　k = 目标维度（4000）

// 对每个训练样本，做一次矩阵乘法
φᵢ = PT gᵢ　　← gᵢ 从 p 维压缩为 k 维

// 压缩效果（BERT 例子）
p = 1.1亿　→　k = 4000　　压缩比 = 27500倍
Φ 大小：44 TB　→　约 1.6 GB　✅ 可以处理了
💡 随机矩阵不需要学习：P 的每个元素直接从正态分布随机采样，没有任何训练过程。这是随机投影神奇的地方——不需要"聪明的"压缩方式，纯随机就足够好。
③ 为什么随机就够了？Johnson-Lindenstrauss 引理
📄 Johnson-Lindenstrauss 引理（1984）核心结论

对于 n 个高维向量，存在一个从 p 维到 k 维的随机线性映射，使得任意两个向量之间的内积以高概率被近似保留。所需的 k 只需满足：

k ≥ C × log(n) / ε²
📝 公式含义：
  • k：压缩后的目标维度
  • n：样本数量
  • ε：允许的误差范围（比如 0.1 = 允许 10% 误差）
  • C：一个常数
  • 🔑 关键：k 只和 log(n) 有关，和原始维度 p 完全无关！

✅ 实际意义：

n = 50000 样本，ε = 0.1（允许10%误差）
→ k ≈ 几千维就够了
→ 不管 p 是 100万还是 1750亿，k 都是这个数！
→ 所以随机投影的压缩维度 k 和模型大小无关，只和数据集大小有关

④ 为什么"保留内积"就够了？

归因公式的核心是：

τ(z)ᵢ ∝ g(z)ᵀ · (ΦᵀΦ)-1 · gᵢ

这个公式里，梯度只以内积的形式出现（gᵢ · gⱼ）。JL 引理保证：

压缩前：gᵢ · gⱼ = x
压缩后：φᵢ · φⱼ ≈ x　（误差在 ε 范围内）
🌍 直觉类比：把1亿个城市在三维地球上的相对距离，投影到二维地图上。两个城市在地球上近，在地图上也近；在地球上远，地图上也远——相对关系被保留了。归因只需要相对关系，不需要绝对坐标，所以压缩后结果基本不变。
⑤ 数字例子：感受内积被保留

设定：原始 4 维，压缩到 2 维，三个样本：橙子A、橙子B（相似）、苹果C（不同）

压缩前的梯度向量和内积：
g₁（橙子A）= [1.0, 0.0, 1.0, 0.0]
g₂（橙子B）= [0.9, 0.1, 0.9, 0.1]
g₃（苹果C）= [0.0, 1.0, 0.0, 1.0]

g₁·g₂ = 1×0.9 + 0×0.1 + 1×0.9 + 0×0.1 = 1.80　← 很相似
g₁·g₃ = 1×0 + 0×1 + 1×0 + 0×1 = 0.00　← 完全不同
随机矩阵 P（4×2）：
P = [[ 0.6, 0.3],
　　 [-0.4, 0.7],
　　 [ 0.5, -0.2],
　　 [ 0.1, 0.8]]
压缩后（φᵢ = Pᵀgᵢ）和内积：
φ₁（橙子A）= [1.10, 0.10]
φ₂（橙子B）= [1.00, 0.14]
φ₃（苹果C）= [-0.30, 1.50]

φ₁·φ₂ = 1.1×1.0 + 0.1×0.14 = 1.114　✅ 仍然很大（原来 1.80）
φ₁·φ₃ = 1.1×(−0.3) + 0.1×1.5 = −0.18　✅ 仍然接近零（原来 0.00）
✅ 结论：橙子A和橙子B压缩后仍然相似，和苹果C仍然不相关。归因分数的排名不变。
⑥ 论文里的实现细节
🎲 随机矩阵不存储

P 的大小 = p × k = 1.1亿 × 4000 ≈ 440 GB，根本存不下。解决方案：用伪随机数生成器实时生成每一行，用完即丢，固定随机种子保证可复现。

⚡ GPU 加速

论文实现了专门的 CUDA kernel，利用 GPU 的 Tensor Core 加速矩阵运算，比朴素实现快 200倍以上。

不同数据集选用的压缩维度 k：
数据集 模型 选用的 k
CIFAR-2 ResNet-9 4,000
CIFAR-10 ResNet-9 20,000
QNLI BERT-base 4,000
ImageNet ResNet-18 15,000
⚠️ k 不是越大越好：论文 Appendix E.1 发现，k 过大反而会降低 LDS 分数。适度压缩有正则化效果，帮助泛化；压缩太少反而引入噪声。

步骤3：估计影响（Newton 步近似）

💡 类比：弹簧平衡

把每个训练样本想象成一个挂在弹簧上的重物，把弹簧"拉"到了平衡位置 θ*。

删掉样本 i = 拿走重物 i。Newton 步近似不等弹簧真的重新稳定，而是根据"重物有多重"和"弹簧有多硬"，直接用公式估算新平衡在哪里。

① 我们要计算什么？

核心问题：删掉训练样本 i，重新训练，模型对测试样本 z 的预测会变多少？

这个变化量就是样本 i 的归因分数——变化越大，说明这个样本越重要。

🐢 暴力方法

对每个样本 i，真的删掉它，重新训练，观察变化。训练集有 n 个样本就要训练 n 次。

n = 50000 → 训练 50000 次 ❌

⚡ Newton 步近似

用一步数学运算估算"删掉样本 i 后参数怎么变"，不需要真的重新训练。

只需计算一次梯度 ✅

② 最终归因公式长什么样？
📄 原文 Equation (13)
τ(z, S) = φ(z)T (ΦTΦ)-1 ΦT Q
📝 每一项的含义：
  • φ(z)：测试样本 z 的（降维后）梯度向量——"测试样本关心哪些参数方向"
  • (ΦTΦ)-1：模型曲率的逆——"哪些参数方向容易被影响"
  • ΦT：所有训练样本梯度矩阵的转置
  • Q：对角矩阵，第 i 项 = (1 − p*ᵢ)，即"模型对样本 i 的不确定程度"
💡 三项合起来的直觉：归因分数高 ↔ 训练样本 i 和测试样本 z 在梯度空间里方向相似，且模型对样本 i 还不够自信，且这个方向是模型容易被推动的方向。
③ p*ᵢ 是什么，怎么算出来的？

p*ᵢ 是模型用训练好的参数 θ* 对训练样本 i 预测为正确类别的概率。完整计算路径：

1
原始输入 xᵢ（一张图片的像素）喂进训练好的神经网络
2
得到模型输出 f(xᵢ; θ*)，一个实数（正数倾向正类，负数倾向负类）
3
乘以标签 yᵢ（+1 或 −1），得到 yᵢ × f(xᵢ; θ*)（正数=预测正确，负数=预测错误）
4
代入 sigmoid 函数：p*ᵢ = 1 / (1 + exp(−yᵢ × f(xᵢ; θ*)))
5
得到 (1 − p*ᵢ)，即不确定程度，用于加权归因分数
💡 直觉：p*ᵢ 接近 1 = 模型对这个样本已经很自信了，删掉它影响不大；p*ᵢ 接近 0.5 = 模型对它还很不确定，删掉影响更大。所以用 (1 − p*ᵢ) 作为权重，不确定的样本归因分数被放大。
④ 数字例子：一步步算出归因分数

设定：二分类（橙子 vs 苹果），模型只有 2 个参数，每个样本只有 2 个特征（颜色值、形状值）

📋 已知条件
θ* = [0.6, 0.4]　　← 训练好的模型参数
样本A（橙子）：x_A = [0.9, 0.8]，y_A = +1
样本B（苹果）：x_B = [0.2, 0.7]，y_B = −1
测试样本z（橙子）：x_z = [0.8, 0.7]
Step 1：计算每个样本的梯度 gᵢ
// 这个极简线性模型里，梯度 = 原始特征向量 xᵢ
g_A = [0.9, 0.8]
g_B = [0.2, 0.7]
g_z = [0.8, 0.7]　← 测试样本的梯度
Step 2：计算模型输出 f(xᵢ; θ*)
f(x_A) = 0.6×0.9 + 0.4×0.8 = 0.54 + 0.32 = 0.86
f(x_B) = 0.6×0.2 + 0.4×0.7 = 0.12 + 0.28 = 0.40
Step 3：计算 p*ᵢ = sigmoid(yᵢ × f(xᵢ))
// 样本A（橙子，y=+1）
p*_A = 1 / (1 + exp(−1 × 0.86))
　　 = 1 / (1 + 0.423) = 0.703　← 预测较准，不确定度低

// 样本B（苹果，y=−1），但 f=0.4>0 说明模型误判为橙子
p*_B = 1 / (1 + exp(−(−1) × 0.40))
　　 = 1 / (1 + exp(−0.40)) 　← 注意：y=−1 所以变成 exp(+0.40)
　　 = 1 / (1 + 1.492) = 0.401　← 预测不准，不确定度高
Step 4：计算不确定程度 (1 − p*ᵢ)
1 − p*_A = 1 − 0.703 = 0.297
1 − p*_B = 1 − 0.401 = 0.599
Step 5：计算归因分数（忽略曲率矩阵，假设各方向同等重要）
τ(z)_A = g_z · g_A × (1−p*_A)
　　　 = (0.8×0.9 + 0.7×0.8) × 0.297
　　　 = (0.72 + 0.56) × 0.297
　　　 = 1.28 × 0.297 = 0.380 ✅ 正分，有帮助

τ(z)_B = g_z · g_B × (1−p*_B)
　　　 = (0.8×0.2 + 0.7×0.7) × 0.599
　　　 = (0.16 + 0.49) × 0.599
　　　 = 0.65 × 0.599 = 0.389 ⚠️ 本应为负分
⚠️ 为什么样本B（苹果）也算出了正分？
因为我们忽略了 (ΦTΦ)-1（曲率矩阵）。它的作用是修正"方向是否真正有帮助"——样本B的 x₂ 特征（形状值 0.7）和测试样本相近，导致内积为正。真实 TRAK 计算中，曲率矩阵会让苹果样本得到负分。
⑤ 归因分数含义总结
归因分数 含义 例子
高正分 删掉这个样本，预测变差很多 同类训练图片，模型还没完全学会
接近零 删不删都无所谓 模型对该样本已非常自信，或与测试样本无关
负分 删掉这个样本，预测反而变好 不同类样本，在"拉着"模型做错误判断

步骤4：集成（为什么要多个模型？）

💡 类比：多个医生会诊

一个医生的判断可能受到随机因素影响（比如那天状态不好）。多个医生独立会诊再综合，结论更可靠。

神经网络训练也有随机性（随机种子、数据顺序、Dropout），同样的训练集训练出来的模型行为会有显著差异。平均多个模型的归因分数能"抹平"这种随机噪声，得到更稳定可靠的结果。

① 为什么训练随机性是个问题？

神经网络训练有多个随机来源：

参数初始化　← 权重随机初始化
数据顺序　　← 每个 epoch 的 batch 顺序随机打乱
Dropout　　← 随机丢弃神经元
数据增强　　← 随机裁剪/翻转图片
😟 单个模型的问题

模型A（种子1）：样本i归因分数 = 0.35
模型B（种子2）：样本i归因分数 = 0.12
哪个才是"真实"影响力？两个都不准。

✅ 集成之后

平均100个模型：样本i归因分数 ≈ 0.23
随机噪声互相抵消，剩下的是稳定的信号。

② 原文的做法：两个关键修改
📄 原文 Section 3.2 Step 4

"To 'smooth out' the impact of such seed-based differences, we aggregate the estimator across multiple trained models, with two small modifications: (a) averaging the two parts of the formula separately before multiplying, and (b) training each model on a random subset of S rather than the full training set."

📝 译文：为了"抹平"随机种子带来的差异，我们在多个训练好的模型上聚合估计量，并做了两个小修改：(a) 先分别平均公式的两个部分，再相乘；(b) 每个模型在 S 的随机子集上训练，而不是全部数据。
③ 修改 (a)：先平均各部分，再相乘

归因公式可以拆成两个独立部分：

τₘ(z) = [ φₘ(z)ᵀ (ΦₘᵀΦₘ)⁻¹ Φₘᵀ ] × [ Qₘ ]
　　　　↑ 特征部分（和测试样本z有关）　　↑ 概率部分（训练集信息）
❌ 朴素方法

对每个模型m：
　算完整 τₘ(z)
最后平均：
　(1/M) Σ τₘ(z)

✅ 论文做法

先分别平均两部分：
　Q̄ = (1/M) Σ Qₘ
　F̄ = (1/M) Σ 特征部分ₘ
最后：τ = F̄ × Q̄

📄 原文 Equation (15)
τM(z) = [ (1/M) Σ φm(z)T(ΦmTΦm)-1ΦmT ] × [ (1/M) Σ Qm ]
📝 原文说这样做的原因是数值稳定性（numerical stability）：每个模型的完整乘积数值可能相差很大，直接平均容易出现数值误差；先在更平滑的矩阵上取平均，再相乘，结果更稳定。
④ 修改 (b)：每个模型用随机子集训练
❌ 朴素方法

模型1：在完整 S 上训练
模型2：在完整 S 上训练
模型M：在完整 S 上训练
（只换随机种子）

✅ 论文做法

模型1：在子集 S₁ ⊂ S 上训练
模型2：在子集 S₂ ⊂ S 上训练
模型M：在子集 Sₘ ⊂ S 上训练
（每个子集随机采样）

💡 为什么子集采样更好？处理冗余样本：
假设训练集里有 100 张极其相似的图片（同一场景）。朴素方法下，每个模型都见过所有 100 张，归因分数被"稀释"，每张图分数都很低。子集采样让每个模型只见到其中一部分，不同模型覆盖不同的图片，平均后整组图片的影响力被正确体现出来。
⑤ 节省计算的两个变体（Appendix E.3）

训练 M 个完全独立的模型成本很高。论文发现两个省钱方案，效果几乎不变：

变体1：同一模型的多个 checkpoint

不训练 M 个独立模型，而是训练少量模型，每个模型保存多个训练过程中的 checkpoint，用这些 checkpoint 代替独立模型。

独立模型数 每模型 checkpoint 数 LDS（CIFAR-10）
520 0.329
6~17 0.340
1010 0.350
100（独立） 1 0.355 ✅ 基准
💡 5 个模型的效果已经接近 100 个独立模型，计算成本降低 20 倍。
变体2：不等训练完全收敛

用早期训练 checkpoint（训练未完成时）来计算梯度，不需要等模型完全收敛。

⑥ 完整集成流程总结
// 准备阶段
随机划分 M 个训练子集 S₁, S₂, ..., Sₘ

// 对每个 m = 1 到 M
① 在 Sₘ 上训练模型 → θ*(Sₘ)
② 计算所有训练样本的降维梯度矩阵 Φₘ
③ 计算概率矩阵 Qₘ（对角线是 (1−p*ᵢ)）
④ 计算特征部分 φₘ(z)ᵀ(ΦₘᵀΦₘ)⁻¹Φₘᵀ

// 集成
Q̄ = (1/M) Σ Qₘ　　　　　　← 平均概率部分
F̄ = (1/M) Σ 特征部分ₘ　　← 平均特征部分

// 最终归因分数
τ_TRAK(z) = F̄ × Q̄

// 后处理（Step 5）
τ_final = S(τ_TRAK; λ̂)　　← 软阈值去噪

步骤5：稀疏化（软阈值处理）

💡 类比：去除背景噪音

归因分数天然是稀疏的——每个测试样本只真正依赖训练集里少数几个样本。但计算过程会引入噪声，让大量"无关样本"也有了微小的非零分数。

软阈值处理把绝对值小于阈值 λ 的分数直接归零，同时把较大的分数向零方向缩小一点，让结果更干净准确。

① 为什么归因分数需要稀疏化？
📄 原文 Section 3.2 Step 5

"Ilyas et al. find that for neural networks attribution scores are often sparse—that is, each test example depends on only a few examples from the training set."

📝 译文：每个测试样本实际上只依赖训练集里少数几个样本，归因分数天然是稀疏的。

但实际计算出来的结果并不稀疏：

// 真实情况（应该稀疏）
τ_真实 = [0, 0, 0.80, 0, 0, 0.30, 0]

// 计算结果（混入了噪声）
τ_计算 = [0.02, -0.01, 0.79, 0.03, -0.02, 0.31, 0.01]
　　　　　↑ 这些是噪声，需要去掉
② 软阈值公式是什么意思？
📄 原文 Equation (16)
S(τ; λ) = (τᵢ − λ) · 1{τᵢ > λ} + (τᵢ + λ) · 1{τᵢ < −λ}
📝 公式做的事情只有三条规则：
  • τᵢ > +λ（正的大值）：结果 = τᵢ − λ　← 保留，但缩小一点
  • τᵢ < −λ（负的大值）：结果 = τᵢ + λ　← 保留，但往零靠近一点
  • −λ ≤ τᵢ ≤ +λ（绝对值小）：结果 = 0　　← 直接归零（去掉噪声）
  • 1{条件} 是指示函数：条件成立 = 1，不成立 = 0

用示意图理解（横轴 = 输入 τᵢ，纵轴 = 输出）：

输出
　|　　　　　　　　　/
　|　　　　　　　　/
+λ|－－－－－－－/
　|　　　　　　/
　|_________/___________________→ 输入 τᵢ
　|　　　　　　\
-λ|－－－－－－－\
　|　　　　　　　　\
　|　　 -λ　　0　　+λ
　　　　 ↑___________↑
　　　　 这个区间内全部归零
③ 软阈值 vs 硬阈值：为什么选软的？
❌ 硬阈值

τᵢ > λ → 完全保留（不变）
τᵢ < λ → 直接归零

在 λ 处有跳跃，函数不连续，数值计算不稳定。

✅ 软阈值

τᵢ > λ → 缩小后保留（τᵢ − λ）
τᵢ < λ → 直接归零

函数连续，在 λ 处平滑过渡，数学性质好。

📚 理论依据：原文引用了 Donoho（1995）的经典论文 "De-noising by soft-thresholding"，这是统计学和信号处理领域处理稀疏信号的标准去噪方法，有近 30 年历史，有严格的最优性证明。
④ λ 怎么选？交叉验证自动确定

λ 不是人工拍脑袋定的，原文用交叉验证自动从数据里选：

1
用 Step 4 集成结果算出初始归因分数 τ_M(z)
2
列出一系列候选 λ 值，例如 {0.001, 0.005, 0.01, 0.05, 0.1, 0.5}
3
对每个候选 λ，用软阈值处理归因分数，得到 τ_λ(z)
4
计算每个 τ_λ(z) 的 LDS 分数（用已有模型评估归因质量）
5
选 LDS 最高的那个 λ，记作 λ̂，用于最终计算
λ 太小
噪声没去干净
LDS 偏低 ❌
λ 刚好 ✅
只去噪声
LDS 最高 🏆
λ 太大
真实信号也丢了
LDS 偏低 ❌
⑤ 数字例子：完整计算一遍

假设 λ̂ = 0.05，集成后归因向量为：

τ_M = [0.02, -0.01, 0.79, 0.03, -0.08, 0.31, -0.04]
τ₁ = 0.02：|0.02| < 0.05 → 归零　　　　→ 0
τ₂ = -0.01：|-0.01| < 0.05 → 归零　　　　→ 0
τ₃ = 0.79：0.79 > 0.05 　→ 0.79 − 0.05 → 0.74 ✅
τ₄ = 0.03：|0.03| < 0.05 → 归零　　　　→ 0
τ₅ = -0.08：-0.08 < -0.05 → -0.08 + 0.05 → -0.03 ✅
τ₆ = 0.31：0.31 > 0.05 　→ 0.31 − 0.05 → 0.26 ✅
τ₇ = -0.04：|-0.04| < 0.05 → 归零　　　　→ 0
τ_TRAK = [0, 0, 0.74, 0, -0.03, 0.26, 0]
噪声清除，只剩样本3（强正影响）、样本5（弱负影响）、样本6（中正影响）
⑥ 最终完整公式（原文 Equation 17）
📄 原文 Equation (17) — TRAK 最终估计量
τTRAK(z, S) := S( [(1/M) Σ φm(z)T(ΦmTΦm)-1ΦmT] × [(1/M) Σ Qm] ; λ̂ )
📝 整个公式的结构：
  • 内层 [ ] × [ ]：Step 4 的集成结果（先平均特征部分和概率部分，再相乘）
  • 外层 S( · ; λ̂ )：Step 5 的软阈值处理（用交叉验证选出的最优 λ̂）
  → 这就是 TRAK 的完整最终输出
