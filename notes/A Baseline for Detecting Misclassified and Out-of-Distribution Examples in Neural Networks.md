
# 1. 文章信息

ICLR 2017

论文首次正式提出将最大 Softmax 概率作为检测异常数据的通用基线


# 2.方法

- 核心基线方法：最大 Softmax 概率（Maximum Softmax Probability, MSP）
    这是文章最核心的贡献。方法非常简单：对于任何输入，直接取模型 Softmax 层输出的最大概率值（即模型预测为Top-1类别的概率）作为“正常分数”。因为模型对见过的、能正确分类的样本给出的最大概率，在统计上总是倾向于**大于**没见过的、预测错误的样本。
    
- 进阶方法：异常检测模块（Abnormality Module）
    为了证明这个 MSP 基线是可以被超越的，作者提出了一种辅助架构。在原有的分类网络上，并联一个辅助解码器（Auxiliary Decoder）用于重建输入图像（或特征）。将原图、重建图以及隐藏层特征拼在一起，喂给一个额外的多层感知机（红色的Abnormality Module），通过二分类训练，专门输出一个评估输入“正常程度”的分数（0到1之间）。

# 3. 流程

1. **训练模型：** 在标准训练集上正常训练一个神经网络分类器。
    
2. **获取输出概率：** 将一个新的测试样本输入到训练好的网络中，经过最后的全连接层和 Softmax 激活函数，得到一个所有类别的概率分布向量。
    
3. **提取特征：** 提取该概率分布向量中的最大值（模型对其预测结果的自信程度）。
    
4. **阈值判断（过滤）：** 设定一个阈值（在实际评估中作者使用了阈值无关的 AUROC 和 AUPR 指标）。
    
    - 如果最大概率 **大于** 阈值，认为是正常样本（同分布 / 预测正确）。
        
    - 如果最大概率 **小于** 阈值，认为是异常样本（OOD数据 / 预测错误），予以过滤或标记。

# 4. 下游任务

1. **计算机视觉（Computer Vision）：**
    
    - **基准数据集：** MNIST, CIFAR-10, CIFAR-100。
        
    - **OOD/异常数据集：** 随机高斯噪声图片、均匀分布噪声图片、SUN场景数据集、Omniglot手写符、notMNIST字体等。
        
2. **自然语言处理（Natural Language Processing）：**
    
    - **情感分类：** IMDB（正常），Customer/Movie Reviews（OOD）。
        
    - **文本分类：** 20 Newsgroups 和 Reuters 新闻数据集的子集划分（部分类别作为正常，部分未见类别作为 OOD）。
        
    - **词性标注（POS Tagging）：** WSJ 新闻语料（正常），Twitter 和 Weblog 语料（OOD）。
        
3. **自动语音识别（Automatic Speech Recognition）：**
    
    - **基准数据集：** TIMIT。
        
    - **OOD/异常数据集：** 带有机场、餐厅、街道等真实背景噪音的 TIMIT 变体，以及中文语音数据集 THCHS-30。

# 5.局限

1. 过度自信： 尽管相对排序有效，但模型对错误分类或 OOD 数据的平均预测概率依然很高（论文表格中显示，错误分类的平均 Softmax 概率高达 70%-80% 以上）。它仅仅是比正确分类稍微低一点点，这导致它在某些分布极度相似的困难样本上区分度不够。
    
2. 仅利用了最后一层信息：Softmax 概率仅仅是模型最后一层 Logits 的变换，它忽略了神经网络深层特征空间中包含的丰富语义和距离信息。（这也是后来为什么学界在它之上提出了马氏距离 Mahalanobis Distance 和能量模型 Energy-based OOD 等改进方法的原因）。
    
3. 不能细粒度区分异常的来源： 这种单一的概率分数无法告诉我们，输入是因为“本身含有噪音”被拒绝，还是因为“属于完全不同的语义类别（如训练集是猫狗，测试集是飞机）”被拒绝。
